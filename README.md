# Angular Handbook & Personal Notes

Welcome to the Angular Handbook repository. This collection contains deep-dive research, practical scenarios, and interview preparation notes for modern Angular development. The content covers the evolution of the framework up through its most modern iterations (Angular 16, 17, and 18), including Standalone Components, Signals, and RxJS/NgRx architectures.

## Table of Contents

### 1. Notebook (`/notebook`)
* **[Angular Deep Research](notebook/angular-deep-research.md)**
  * Comprehensive notes on the architectural shift to Standalone Components, the reactivity revolution with Signals, modern control flow (`@if`, `@for`, `@defer`), Server-Side Rendering (SSR) hydration, and NgRx SignalStore.

### 2. Personal Notes (`/persnoal_notes`)
* **[The Ultimate Angular Personal Handbook](persnoal_notes/angular-complete-journey.md)**
  * A complete guide covering the evolution from AngularJS to Angular 2+, core building blocks (components, directives, pipes), Dependency Injection, routing, reactive forms, RxJS, and modern features of the "Angular Renaissance".
* **[Full-Stack Interview Scenarios: Angular with Java & Node.js](persnoal_notes/full-stack-angular-java-node-scenarios.md)**
  * In-depth full-stack logic covering JWT authentication flows, CORS security, refresh tokens, Cross-Site Request Forgery (CSRF) mitigation, and real-time interactions with WebSockets.
* **[Practical Mastery: RxJS, NgRx & AI Integration Scenarios](persnoal_notes/practical-rxjs-ngrx-ai-scenarios.md)**
  * Code-heavy practical implementations for handling LLM/AI endpoints: handling race conditions, polling long-running background jobs, and using the modern NgRx SignalStore for flawless state management.
* **[Product-Based Company HTTP Interview Scenarios](persnoal_notes/product-based-http-interview-scenarios.md)**
  * The most frequently tested HTTP/RxJS scenarios: parallel API calls (`forkJoin`), sequential dependent requests (`switchMap`), global error handling via Interceptors, retry mechanisms for flaky networks, and data caching (`shareReplay`).
